About SOAP

Simple Object Access Protocol (SOAP) is a messaging protocol specification for exchanging structured information in the implementation of web services in computer networks. Its purpose is to induce extensibility, neutrality and independence. It uses XML Information Set for its message format, and relies on application layer protocols, most often Hypertext Transfer Protocol (HTTP) or Simple Mail Transfer Protocol (SMTP), for message negotiation and transmission. About JAX-WS

Java API for XML Web Services (JAX-WS) is an essential part of Java EE for building a web service. The JAX-WS 2.2 specification JSR 224 defines a standard Java- to-WSDL mapping which determines how WSDL operations are bound to Java methods when a SOAP message invokes a WSDL operation. About The Demo

This demo will ilustrate how SOAP messages are being transacted. This web service contains an encapsulation of simple id-name table in a realtime database. They are six (out of seven) operations supported in web service that simulated in this demo using client-side Javascript. We will see what are the XML request data from your computer and the XML response data from the web service that establishing a SOAP communication.

Here is the endpoint of the web service. From there you can download the WSDL file so you can access the web service with your own client. (Please behave yourself while using it)
Note : Because i put this web service in a free-tier Heroku cloud. In the first time you access the web service, it will be likely in sleep mode. The first request will be taking about 30 - 60 seconds to "wake up" the web service, then a later request will behave normally. Sorry for the inconvenience.
